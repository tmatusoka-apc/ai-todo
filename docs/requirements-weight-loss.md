# 減量特化サービス 要件定義書

## 1. サービスコンセプト

### 1.1 ターゲットユーザー
- ITエンジニア（20〜40代）
- 在宅・リモートワーカー
- 運動不足・長時間デスクワークに悩む社会人
- ダイエット初心者〜再挑戦者

### 1.2 サービスの目標
- **月2〜3kgの無理のない減量**を実現する（急激な制限なし）
- デスクワーカーの生活習慣に合わせた現実的なアプローチ
- 習慣化しやすい小さなタスクで継続率を高める

### 1.3 サービスの特徴
- **減量ToDo**: 初心者向けの具体的な日次タスク（運動/食事/習慣カテゴリ）
- **NotebookLLM風ナレッジ蓄積**: 自分の減量メモを貯めてパーソナルな知識ベースを構築
- **AIチャットボット**: 個人ナレッジ + Gemini API による RAG-lite チャット

## 2. 機能要件

### 2.1 体重ログ
- **日次体重記録**: 日付・体重（kg）を入力して記録
- **BMI自動計算**: プロフィールの身長から BMI を自動算出
- **グラフ表示**: 過去30日分の折れ線グラフ（Canvas API）
- **目標体重ライン**: グラフ上にオレンジ点線で目標体重を表示
- **記録一覧**: テーブル形式で一覧表示・削除機能

### 2.2 減量ToDo
- **タスク作成**: タイトル・カテゴリ（運動/食事/習慣）・日付を指定
- **完了トグル**: チェックボックスで完了/未完了を切り替え
- **カテゴリフィルタ**: 全て・運動・食事・習慣 でフィルタリング
- **スターターキット**: 初回起動時に5つのデフォルトタスクをバッチ作成
  - 今日の体重を記録する（習慣）
  - 水を8杯飲む（食事）
  - 20分ウォーキングする（運動）
  - 夜食を控える（21時以降は食べない）（食事）
  - 7時間以上睡眠を取る（習慣）
- **削除機能**: 不要なタスクを削除
- **定期タスクと不定期タスク**: タスクは「定期タスク」（recurring）と「不定期タスク」に分類される。定期タスクは完了した翌日に自動で未完了状態へリセットされ、毎日繰り返し取り組める。不定期タスク（一度きりのタスク）は完了後もそのまま完了状態を保持する

### 2.3 ナレッジベース（NotebookLLM風）
- **記事・メモ入力**: タイトル・本文・タグ（スペース区切り）で登録
- **タグサジェスト**: よく使うタグを候補表示（#食事制限, #運動, #代謝, #睡眠, #水分, #外食, #デスクワーク）
- **カード一覧表示**: タイルレイアウトで見やすく表示
- **検索フィルタ**: タイトル・タグ・本文でリアルタイム検索
- **編集・削除**: 登録済みナレッジの更新・削除
- **プリセットテンプレート**: トグルボタンで「ダイエット」「トレーニング」「サプリ」「生活習慣」の4種類のプリセットテンプレートを表示/非表示でき、テンプレートを選択することでナレッジ登録をすばやく行える

### 2.4 AIチャットボット
- **RAG-lite チャット**: 個人のナレッジベース + Gemini API で個人化された回答
- **簡潔な回答**: AIの回答は約300字以内に制限し、要点を簡潔に提示する
- **ナレッジ参照明示**: 関連するナレッジがある場合はそのタイトルを挙げて参照を案内する
- **吹き出しUI**: ユーザー右側・AI左側の会話形式
- **ローディング表示**: 「AIが考え中...」アニメーション
- **チャット履歴**: セッション内メモリ（永続化不要）
- **ToDoへの追加**: 各AI回答には「📝 ToDoに追加」ボタンが付き、回答テキストを番号付き/箇条書きリストで分割して最大5件の個別チェック可能なToDo項目に変換し、選択したものだけをToDoリストに追加できる

### 2.5 進捗ダッシュボード
- **目標体重設定**: プロフィールから目標体重・目標期間を設定
- **現在体重**: 最新の記録体重を表示
- **達成率**: （開始体重 - 現在体重）/（開始体重 - 目標体重）× 100%
- **ストリーク**: 連続体重記録日数
- **今日のToDo進捗**: 完了数/総数をプログレスバーで表示
- **最近のナレッジ数**: 追加したナレッジ数

## 3. 非機能要件

### 3.1 認証
- 既存の Amazon Cognito 認証を共有
- JWT（idToken）を Authorization ヘッダーで送信
- 期限切れ時は refresh token で自動更新

### 3.2 インフラ
- **バックエンド**: AWS Lambda (Node.js 20) + Amazon DynamoDB + Amazon API Gateway
- **フロントエンド**: Amazon S3 + Amazon CloudFront
- **AI**: Google Gemini API（Secrets Manager経由）

### 3.3 UI/UX
- **カラーテーマ**: グリーン系（#22c55e, #16a34a）- 健康・減量イメージ
- **レスポンシブ**: モバイルファースト対応
- **UX**: ローディングスピナー、トースト通知、スムーズなアニメーション
- **シングルファイル**: `weight-loss.html` 1ファイルで完結（SPA）

### 3.4 セキュリティ
- 全APIエンドポイントに Cognito 認証必須
- CORS ヘッダーを全レスポンスに付与
- DynamoDB の PK に userId を使いユーザー間のデータ分離

### 3.5 パフォーマンス
- DynamoDB: PAY_PER_REQUEST（オンデマンド課金）
- Lambda: timeout 60秒, memorySize 512MB
- PointInTimeRecovery: true（全テーブル）
- RemovalPolicy: RETAIN（全テーブル）
