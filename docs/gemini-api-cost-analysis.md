# Gemini API 利用申請・コスト試算ドキュメント

## 1. ツール概要

**AI タスク管理・習慣形成支援アプリ** の開発。

ユーザーが日々のタスク・習慣を登録・管理しながら、AI によって個人の行動実績に基づいたパーソナライズされた提案を受け取ることができる。継続的な習慣形成と目標達成をサポートすることを目的とする。

---

## 2. Gemini API が必要な理由

以下の機能において、ユーザーごとに異なる文脈を踏まえた **自然言語での生成** が必要なため。

| 機能 | 内容 |
|------|------|
| デイリータスク提案 | 今週の活動実績（運動・食事・睡眠の達成率）を分析し、今日取り組むべき1件のタスクを AI が提案 |
| 週次タスク提案 | カテゴリ（運動・食事・睡眠）ごとに週次の推奨タスクを自動生成 |
| 目標バックキャスト | ユーザーが設定した長期目標・達成期限から逆算し、週ごとの具体的な行動計画を AI が自動生成 |
| チャットサポート | ユーザーからの質問に対して、個人の記録・目標・知識ベースを踏まえた回答を生成 |

---

## 3. 利用モデル・料金単価

| 項目 | 値 |
|------|-----|
| モデル | `gemini-2.5-flash-lite` |
| 入力トークン単価 | $0.10 / 1,000,000 トークン |
| 出力トークン単価 | $0.40 / 1,000,000 トークン |

> ※ モデル名を `gemini-2.0-flash` から `gemini-2.5-flash-lite` に更新。料金単価は旧モデル時点のものを引用しており、`gemini-2.5-flash-lite` の正式単価で再確認・更新が必要。

> **選定理由**: 応答速度・コストのバランスが優れており、タスク提案・チャット用途に適している。

## 5. コスト試算（MAX値）

### 5-1. 1ユーザー・1日あたりのトークン数

| | 入力トークン | 出力トークン |
|--|------------|------------|
| デイリー提案 | 300 | 512 |
| 週次提案（日割り） | 50 | 73 |
| バックキャスト（日割り） | 12 | 30 |
| チャット × 5回 | 4,000 | 5,120 |
| **合計** | **4,362** | **5,735** |

### 5-2. 利用人数別・月次コスト試算

計算式:
```
月次入力コスト  = 入力トークン/日 × 30日 × 人数 × $0.10 / 1,000,000
月次出力コスト  = 出力トークン/日 × 30日 × 人数 × $0.40 / 1,000,000
月次合計コスト  = 入力コスト + 出力コスト
```

| 利用人数 | 月次入力トークン | 月次出力トークン | 月次コスト（USD） | 月次コスト（JPY）|
|---------|--------------|--------------|----------------|----------------|
| 10名 | 1,309,000 | 1,721,000 | $0.13 + $0.69 = $0.82 | 約 123円 |
| 30名 | 3,926,000 | 5,162,000 | $0.39 + $2.06 = $2.45 | 約 368円 |
| 50名 | 6,544,000 | 8,603,000 | $0.65 + $3.44 = $4.09 | 約 614円 |
| 100名 | 13,087,000 | 17,205,000 | $1.31 + $6.88 = $8.19 | 約 1,228円 |

> ※ 為替レート: 1 USD = 150 円で換算  
> ※ 全ユーザーが毎日フル活用した場合のMAX値。実運用ではチャット未使用のユーザーも多く、さらに低くなる見込み。  
> ※ **100名がフル活用しても月額 1,500 円以内に収まる**。

## 6. 予算超過時の自動停止の仕組み

クレジットが枯渇すると Gemini API は自動的に `429 RESOURCE_EXHAUSTED` を返す。  
さらに以下のいずれかのアプローチで **青天井の課金を防止** できる。

### アプローチ A：予算アラート + Cloud Run 関数（公式推奨）

```
Google Cloud 予算アラート（例: $10到達）
  └── Pub/Sub に通知
        └── Cloud Run 関数が起動
              └── Cloud Billing API を呼び出してプロジェクトの請求を無効化
```

- プログラムを一度作れば確実にシャットダウン可能
- Google Cloud 公式ドキュメントでも案内されている方法

### アプローチ B：API の割り当て（Quota）上限を設定（より簡単・推奨）

- Google Cloud コンソールの「割り当て（Quotas）」から **1日あたりの最大リクエスト数** を設定
- 上限に達すると自動的に `429 Too Many Requests` が返る
- アプリ側のフォールバック機能（ルールベース固定提案）が自動で作動
- プログラム不要で設定だけで完結

### 現在のアプリのフォールバック動作

| 状況 | 動作 |
|------|------|
| API 正常 | Gemini の回答を返す |
| 429 / クォータ超過 | ルールベースの固定提案・固定メッセージを返す |
| その他エラー | フォールバックメッセージを表示 |

---

## 7. ルール・上限設定（推奨）

コスト管理・リスク低減のため以下のルールを設ける。

| ルール | 内容 |
|--------|------|
| チャット上限 | 1ユーザー・1日あたり **10回** まで（現状MAXで5回想定） |
| モデル固定 | `gemini-2.5-flash-lite` を使用（上位モデルへの自動昇格なし） |
| Quota 設定 | Google Cloud コンソールで1日あたりのリクエスト上限を設定 |
| フォールバック | API 障害・クォータ超過時はルールベースの固定提案に自動切り替え |
| 利用フェーズ | 社内検証フェーズ（数名〜100名規模）、本番展開前に再試算 |

---

## 8. データ取り扱い

| 項目 | 内容 |
|------|------|
| 送信するデータ | 行動実績の統計データ（達成率・習慣カテゴリ）、チャットメッセージ本文 |
| 送信しないデータ | 氏名・メールアドレス等の個人識別情報 |
| データ保持 | Google AI Studio の有料枠または Vertex AI を利用することで、入力データが学習に利用されない。リクエスト完了後はデータを保持しない（ステートレス） |

---

## 9. まとめ

- **想定利用人数 100名・全員がフル活用した場合でも月額約 1,228 円（MAX値）**
- Gemini API はクレジット枯渇時に自動で `429` を返すため **青天井の課金は発生しない**
- さらに Quota 上限設定または予算アラート + Cloud Run 関数で **二重の安全網** を構築可能
- フォールバック機能により API 停止時もサービスは継続稼働
- モデル固定・上限ルールにより予期せぬコスト増加を防止
